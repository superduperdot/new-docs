## 1. Sign up for Brale

To [sign up](https://app.brale.xyz/signup) for a Brale account, you will need to complete KYB before you are approved for API access.

## 2. Create API credentials (mainnet vs testnet)

Create an application on the [Settings page](https://app.brale.xyz/settings?tab=api) in the Dashboard.

![Image](/images/brale/settings-api-credentials-dark@2x.png)

Select **Mainnet** or **Testnet** and enable required resources. Use testnet while integrating; switch to mainnet only when ready to move value.

![Image](/images/brale/settings-api-create-application-dark@2x.png)

Brale will make your API key available only once. Please save the key in a secure location.

## 3. Authenticate (client_credentials, short-lived tokens)

Brale uses OAuth2 with the `client_credentials` grant type for secure access to our APIs. Tokens are short-lived (~60 minutes); refresh automatically before expiry using `expires_in`. The Authorization header uses Basic HTTP Authentication and requires encoding your `client_id` and `client_secret` into a Base64 format.

To generate the Base64 encoded string, you can use a command-line tool like base64 or an online encoder. Ensure that your `client_id` and `client_secret` are separated by a colon `:` and then encoded. This encoded string will be used in the Authorization header.

```bash
curl --request POST \
  --url https://auth.brale.xyz/oauth2/token \
  --header 'Authorization: Basic ${BASE_64_OF(client_id:client_secret)}' \
  --header 'Content-Type: application/x-www-form-urlencoded' \
  --data grant_type=client_credentials
```

Using the Brale API Postman collection, you can configure the `client_id` and the `client_secret` environment variables that are used to authenticate against the API.

![Image](/images/brale/brale-api-postman-collection-variables-dark@2x.png)

Once these variables are added, you can request an Auth token from the Authorization tab.

![Image](/images/brale/brale-api-postman-authorization-dark@2x.png)

## 4. Get your Account ID (KSUID)

Now fetch your account ID that you will pass into subsequent requests. IDs are KSUIDs (26 chars, time-sortable); do not use UUIDs. Since you haven't registered any customer accounts yet, the only account ID returned will be your own account.

**GET** `https://api.brale.xyz/accounts`

```bash title="Request"
curl --request GET \
  --url https://api.brale.xyz/accounts \
  --header "Authorization: Bearer ${AUTH_TOKEN}"
```

```json title="Response"
{
  "accounts": [
    "2Js1YFqlfxgNqC2KTPEjrWIwKU7" // Your account ID
  ]
}
```

## 5. Get your Address ID (internal custodial)

Fetch addresses for your account. `address_id` is the universal source/destination primitive for transfers (covers on-chain wallets and fiat endpoints).

:::info[Path Rule]
Every endpoint that contains `{account_id}` must be called with the value you fetched in Step 4, e.g.
`GET` `https://api.brale.xyz/accounts/2Js1YFqlfxgNqC2KTPEjrWIwKU7/addresses`. Do not send the account ID in the body or query string.
:::

## 6. Create your first transfer (Idempotency-Key required)

Use Transfers to initiate funds movement from fiat to stablecoins (onramp), stablecoins to fiat (offramp), or stablecoins to stablecoins. Always send a unique `Idempotency-Key` per logical transfer and reuse it on retries of the same transfer; never send it on GET requests.

Here's an example of onramping USD to SBC on Base via wire transfer. Include the account ID and the address ID you fetched in Step 4. You must also include an idempotency key.

**POST** `https://api.brale.xyz/accounts/your_account_id/transfers`

```bash title="Request"
curl --request POST \
  --url "https://api.brale.xyz/accounts/${ACCOUNT_ID}/transfers" \
  --header "Content-Type: application/json" \
  --header "Authorization: Bearer ${AUTH_TOKEN}" \
  --header "Idempotency-Key: $(uuidgen)" \          # generate a fresh key for each transfer
  --data '{
    "amount":    { "value": "10", "currency": "USD" },
    "source":    { "value_type": "USD", "transfer_type": "wire" },
    "destination": {
      "address_id": "'"${ADDRESS_ID}"'",
      "value_type": "SBC",
      "transfer_type": "base"
    }
  }'
```

The response will include the transfer, which is opened in a pending state and transitions to processing once fiat funds are received.

```json title="Response"
{
  "id": "2xNL6PAF0cbcQHyjMQJ2RKRfbD9",
  "status": "pending",
  "source": {
    "value_type": "USD",
    "transfer_type": "wire"
  },
  "destination": {
    "address_id": "2VcUIonJeVQzFoBuC7LdFT0dRe4",
    "value_type": "SBC",
    "transfer_type": "base"
  },
  "updated_at": "2025-05-20T20:36:48.147281Z",
  "created_at": "2025-05-20T20:36:48.147281Z",
  "amount": {
    "value": "1",
    "currency": "USD"
  },
  "note": null,
  "wire_instructions": {
    // Brale Bank instructions
  }
}
```

## 7. Poll and reconcile

- Poll `GET /accounts/{account_id}/transfers` using pagination tokens; store `id`, `idempotency key`, timestamps, status, and provider references for reconciliation.
- On 401, refresh the token and retry idempotently with the same `Idempotency-Key`.
