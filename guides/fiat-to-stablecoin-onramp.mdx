title: "Fiat to Stablecoin (Onramp)"
---

<p className="intro-subheading">
Brale makes it easy for you and your customers to convert USD into stablecoins via wire transfer or ACH.
</p>

<Warning>
**Fiat rails (wire, ACH) require mainnet.** The wire and ACH flows in this guide cannot be fully executed on testnet because they require real bank infrastructure. You can validate the automation creation flow and API shape on testnet, but actual fiat deposits require mainnet credentials and a funded bank account.
</Warning>

There are two ways to onramp to stablecoins through the Brale API:

- Submit a [Transfer](/key-concepts/transfers) request with a source `transfer_type` of `wire` or `ach_debit`.
- Create an [Automation](/key-concepts/automations), which automatically mints stablecoins to a pre-configured destination wallet when funds are received (Automatic Onramp).

Businesses can also onramp directly via the Brale Dashboard, and individuals can onramp through [Brale.io](https://brale.io/).

## Prerequisites

- Bearer token ([How to authenticate](/key-concepts/authentication))
- Your `account_id` ([How to get your account_id](/guides/how-to/get-your-account-id))
- An internal (custodial) `address_id` for the stablecoin destination ([How to get address_ids](/guides/how-to/get-your-address-ids))
- **For ACH debit**: A Plaid-linked address ([Addresses — Plaid integration](/key-concepts/addresses))

## Fiat to Stablecoin via Wire

**POST** `https://api.brale.xyz/accounts/{account_id}/transfers`

```bash
curl --request POST \
  --url "https://api.brale.xyz/accounts/${ACCOUNT_ID}/transfers" \
  --header "Authorization: Bearer ${ACCESS_TOKEN}" \
  --header "Content-Type: application/json" \
  --header "Idempotency-Key: $(uuidgen)" \
  --data '{
    "amount": {
      "value": "100",
      "currency": "USD"
    },
    "source": {
      "value_type": "USD",
      "transfer_type": "wire"
    },
    "destination": {
      "address_id": "'${ADDRESS_ID}'",
      "value_type": "SBC",
      "transfer_type": "solana"
    }
  }'
```

```json title="Response (201)"
{
  "id": "2xNL6PAF0cbcQHyjMQJ2RKRfbD9",
  "status": "pending",
  "source": {
    "value_type": "USD",
    "transfer_type": "wire"
  },
  "destination": {
    "address_id": "2VcUIonJeVQzFoBuC7LdFT0dRe4",
    "value_type": "SBC",
    "transfer_type": "solana"
  },
  "amount": {
    "value": "100",
    "currency": "USD"
  },
  "wire_instructions": {
    "bank_name": "...",
    "routing_number": "...",
    "account_number": "...",
    "beneficiary_name": "..."
  },
  "created_at": "2025-05-20T20:36:48.147281Z",
  "updated_at": "2025-05-20T20:36:48.147281Z",
  "note": null
}
```

The transfer starts as `pending`. Once Brale receives the wire deposit, stablecoins are minted to the destination address and the transfer transitions to `complete`.

## Fiat to Stablecoin via ACH Debit

Initiate ACH debits from Plaid-linked addresses. See [Addresses](/key-concepts/addresses) for the Plaid linking flow.

**POST** `https://api.brale.xyz/accounts/{account_id}/transfers`

```json title="Request"
{
  "amount": {
    "value": "100",
    "currency": "USD"
  },
  "source": {
    "address_id": "${PLAID_LINKED_ADDRESS_ID}",
    "value_type": "USD",
    "transfer_type": "ach_debit"
  },
  "destination": {
    "address_id": "${INTERNAL_ADDRESS_ID}",
    "value_type": "SBC",
    "transfer_type": "solana"
  }
}
```

## Fetching Transfers

**GET** `https://api.brale.xyz/accounts/{account_id}/transfers/{transfer_id}`

```bash
curl --request GET \
  --url "https://api.brale.xyz/accounts/${ACCOUNT_ID}/transfers/${TRANSFER_ID}" \
  --header "Authorization: Bearer ${ACCESS_TOKEN}"
```

## Fiat to Stablecoin Automation (Automatic Onramp)

Automations provide a unique account number and routing number that accepts wire transfers and ACH deposits, and automatically mints stablecoins to a pre-defined wallet address.

**POST** `https://api.brale.xyz/accounts/{account_id}/automations`

```bash
curl --request POST \
  --url "https://api.brale.xyz/accounts/${ACCOUNT_ID}/automations" \
  --header "Authorization: Bearer ${ACCESS_TOKEN}" \
  --header "Content-Type: application/json" \
  --header "Idempotency-Key: $(uuidgen)" \
  --data '{
    "name": "XYZ Onramp",
    "source": {
      "transfer_type": null,
      "value_type": "USD"
    },
    "destination": {
      "address_id": "'${ADDRESS_ID}'",
      "value_type": "SBC",
      "transfer_type": "solana"
    }
  }'
```

<Note>
The source `transfer_type` is `null` because the automation accepts funds via any supported fiat rail (wire or ACH). The rail is determined by how the depositor sends funds, not by the API caller.
</Note>

When an automation becomes `active`, the `source.funding_instructions` object is populated with virtual bank account details:

**GET** `https://api.brale.xyz/accounts/{account_id}/automations/{automation_id}`

```json title="Response (200)"
{
  "id": "3670hdMDFjH6CzQrOoYWQGPssrH",
```

## Verification

The following can be verified on testnet:

| Step | Testnet? | Notes |
|------|----------|-------|
| Create automation | Yes | Automation is created and transitions to `active` |
| Read `funding_instructions` | Yes | Virtual bank details are populated |
| List/get automations | Yes | CRUD operations work |
| Wire deposit → mint | **No (mainnet only)** | Requires real wire to virtual account |
| ACH debit → mint | **No (mainnet only)** | Requires Plaid + real bank account |

```bash
# Testnet: List existing automations — expect HTTP 200
curl -s -w "\nHTTP: %{http_code}" \
  "https://api.brale.xyz/accounts/${ACCOUNT_ID}/automations" \
  --header "Authorization: Bearer ${ACCESS_TOKEN}"

# Testnet: Get a specific automation with funding_instructions
curl -s "https://api.brale.xyz/accounts/${ACCOUNT_ID}/automations/${AUTOMATION_ID}" \
  --header "Authorization: Bearer ${ACCESS_TOKEN}"
```
